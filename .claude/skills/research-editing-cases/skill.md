# Research Editing Cases — 기사 수정 사례 조사 스킬

신문 기사의 수정 사례를 체계적으로 웹 조사하여 수집하는 스킬.

## 조사 방법

### 1. 검색 전략

다음 검색어 조합으로 WebSearch를 수행한다:

**표현/문장 영역:**
- "신문 기사 교열 사례"
- "기자 맞춤법 오류"
- "기사 문장 수정 전후"
- "신문 비문 사례"
- "기사 중복표현"
- "언론 문장 교정"

**구조/형식 영역:**
- "기사 리드 작성법"
- "역피라미드 구조 사례"
- "기사 제목 수정"
- "인용문 처리 오류"
- "기사 구성 실수"

**정확성/윤리 영역:**
- "정정보도 사례"
- "언론중재위원회 결정"
- "기사 팩트체크 실패"
- "언론 윤리 위반 사례"
- "차별 표현 가이드라인 언론"

### 2. 사례 수집 기준

각 사례는 반드시 다음을 포함한다:
- **수정 전(Before)**: 원래 기사의 문제 부분
- **수정 후(After)**: 수정된 올바른 표현/구조
- **수정 이유**: 왜 수정이 필요한지 근거
- **출처**: 정보의 출처 (URL, 저자, 발행일)

### 3. 출처 우선순위

1. 신문사 공식 스타일 가이드 / 교열 담당자 칼럼
2. 언론학 교수/연구자의 논문/기고
3. 한국기자협회, 언론중재위원회 등 공식 기관 자료
4. 국립국어원 어문 규정/한글 맞춤법
5. 언론 모니터링 단체 보고서
6. 현직/전직 기자의 블로그/칼럼

### 4. 카테고리 분류

수집된 사례를 다음 카테고리로 분류한다:

| 대분류 | 소분류 |
|--------|--------|
| 표현/문장 | 맞춤법·띄어쓰기, 비문, 중복 표현, 관용구 오용, 문체, 피동·사동, 조사 오용 |
| 구조/형식 | 리드, 역피라미드, 제목, 인용문, 단락 구성, 기사 유형별, 캡션 |
| 정확성/윤리 | 사실 오류, 수치 오용, 출처 표기, 편향, 인권·차별, 선정성 |

### 5. 산출물

각 리서처는 자신의 영역 산출물을 `_workspace/` 디렉토리에 마크다운으로 저장한다.
